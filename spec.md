# Спецификация проекта: Платформа ИИ-тестирования сайтов

## 1. Обзор проекта
Веб-приложение для автоматизированного многофакторного тестирования сайтов с использованием искусственного интеллекта. Система позволяет проводить аудит производительности, UX/UI, доступности и функциональности с генерацией детальных отчетов.

## 2. Технологический стек
- **Backend**: FastAPI (Python) — асинхронный фреймворк для API.
- **Frontend**: React (TypeScript) + Tailwind CSS — современный UI.
- **Database & Storage**: Supabase (PostgreSQL) — хранение данных пользователей, результатов тестов, скриншотов и видео.
- **AI Integration**: LiteLLM — единый интерфейс для работы с различными LLM (OpenAI, Anthropic, локальные модели по IP).
- **Browser Automation**: Playwright — захват скриншотов, видео и имитация действий пользователя.
- **Reports**: ReportLab или подобные библиотеки для генерации PDF.

## 3. Архитектура (Layered Architecture)
Следование принципам чистой архитектуры:
- `app/api`: Маршруты (routes), валидация входящих данных (Pydantic схемы).
- `app/service`: Бизнес-логика, координация между доменом и внешними сервисами (Playwright, LiteLLM, Supabase).
- `app/domain`: Ядро системы, модели данных, бизнес-правила и логика тестов.
- `app/infrastructure`: Клиенты для внешних API (Supabase, LiteLLM), конфигурации.

## 4. Основные функции

### 4.1. Панель управления (Dashboard)
- Поле ввода URL.
- Выбор уровня проверки:
    - **Экспресс**: Быстрая проверка главных страниц, базовые метрики.
    - **Стандарт**: Глубокий анализ основных путей пользователя, проверка форм.
    - **Глубокий**: Полное сканирование всех ссылок, анализ анимаций, стресс-тесты.

### 4.2. Интеграция ИИ-моделей
- Добавление кастомных моделей через LiteLLM (указание API Base URL/IP, API Key, Model Name).
- Использование ИИ для анализа визуальных ошибок и логических несоответствий на страницах.

### 4.3. Настройка сложности
Три режима конфигурации:
1. **Быстрый**: Пресеты настроек.
2. **Сбалансированный**: Настройка глубины и типов проверок (SEO, Performance, UI).
3. **Индивидуальный**: Полный контроль (скорость запросов, глубина вложенности, конкретные сценарии Playwright).

### 4.4. Отчетность и медиа
- **Ошибки**: Автоматический скриншот проблемного места.
- **Анимации**: Запись видео при обнаружении лагов или некорректных переходов.
- **PDF-отчет**: Текстовое описание проблем + визуальные доказательства (фото).

### 4.5. Личный кабинет
- История всех запусков тестов.
- Статус текущих проверок.
- Ссылки на скачивание PDF и просмотр результатов в браузере.

### 4.6. Уведомления
- **Telegram**: Бот-уведомления о завершении или критических ошибках.
- **Email**: Итоговое письмо со ссылкой на отчет.

## 5. Модель данных (Supabase)
- `users`: Профили пользователей.
- `tests`: Информация о тестах (URL, уровень, статус).
- `test_results`: Детальные данные по каждому тесту (JSON лог).
- `media`: Ссылки на Storage для скриншотов и видео.
- `ai_models`: Конфигурации подключенных моделей пользователя.

## 6. Безопасность
- JWT аутентификация (Supabase Auth).
- Изоляция процессов Playwright в контейнерах (рекомендуется для продакшена).
- Шифрование API ключей для сторонних ИИ-моделей.
